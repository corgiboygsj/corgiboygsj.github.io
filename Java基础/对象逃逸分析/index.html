Java对象是否都分配到堆上？答案是否定的，通过逃逸分析可以将不会被外界引用的对象分配在栈上，可以随着栈一起被回收，同时减少了堆的压力也就减少了gc次数，达到了优化程序执行时间的目的。

什么是逃逸分析

逃逸分析通过分析对象的作用域来判断对象是否会被分配方法之外的地方所引用和持有。逃逸分析可以有效的减少线程同步开销和堆内存分配的压力。通过分析如果发现了没有发生逃逸的对象，则可以将对象直接分配到栈上，而不是分配到堆上，这样可以让对象随着线程栈一起被释放，可以减少堆的内存占用以及触发垃圾回收的次数和时间的开销。

逃逸分析可以做什么

  • 锁消除
  • 将对象分配到栈上
  • 分配对象的标量替换

锁消除

Java6中synchronized的锁优化就是使用到了逃逸分析，通过判断synchronized的作用范围、代码块中是否有多线程竞争条件、锁对象是否是共享的等条件判断是否可以不加synchronized。

对象分配到栈上

分析方法中的对象如果不会发生逃逸，则可以讲对象直接分配到栈上，让其随着线程栈的释放而释放，达到减少堆内存回收压力以及gc次数的目的，可以优化性能。

对象标量替换

标量是java中无法继续拆解的最小数据，比如基本数据类型就是标量。而对象这种通过基本数据类型进行聚合的就可以拆分成很多的标量。如果发现对象没有发生逃逸，对象不会被外界访问的话则可以将对象拆分成标量。

经过逃逸分析，可以将Point对象拆分为x和y两个标量。
