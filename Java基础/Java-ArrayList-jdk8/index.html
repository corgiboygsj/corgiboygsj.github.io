fail-fast原则

fail-fast机制是一种当检测出并发更改结构时抛出异常的处理机制，主要体现在通过迭代器遍历过程中对集合结构进行了更改抛出并发修改异常，主要会出现在两种情况之下。

  • 单线程下遍历集合时对结构进行了更改。
  • 多线程下迭代器遍历过程中其他线程对结构进行了更改。

fail-fast机制不能保证一发生并发修改就立刻抛出异常，只能保证在迭代器遍历过程中检测出是否发生了结构的变化。

遍历集合时删除/添加元素出现ConcurrentModificationException

遍历集合时准确说是使用迭代器遍历集合时删除或添加元素会出现ConcurrentModificationException异常。
遍历集合的四种方式以及出现异常的情况

  • for
    通过for遍历集合的时候调用集合的删除/添加不会出现ConcurrentModificationException。
  • foreach（底层使用的就是迭代器）
    通过foreach遍历结合时调用集合的删除/添加方法会出现ConcurrentModificationException。
  • iterator
    通过迭代器遍历集合时调用集合的删除/添加方法会出现ConcurrentModificationException，但是调用迭代器的删除/添加并不会出现异常。
  • list.foreach
    java8提供的遍历方式，遍历过程中如果调用集合的添加/删除方法会出现ConcurrentModificationException。

原因就是fail-fast机制，在迭代器遍历时候不可以使用集合的删除/新增对集合结构进行改变，但是可以使用迭代器提供的方法进行操作。Java8的list.foreach也会在代码中判断modCount和expectedModCount是否相等进行并发修改判断。

所以在迭代器遍历时要是使用迭代器提供的增删方法。

适用场景

ArrayList是数组结构，适合随机读取、读多写少的情况，不适合频繁变动元素位置或者在集合中间位置增删元素。
