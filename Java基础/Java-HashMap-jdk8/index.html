HashMap是平时使用非常广泛的集合工具之一。其内部的数据结构是哈希表，哈希表的查找效率是O（1），效率很高。本文通过增删改查的顺序阅读源码，并且分析了一下并发情况的问题和其他的拓展。JDK版本是1.8。

HashMap数据结构

解决哈希冲突的方式

  • 开放定址法
  • 链地址法

HashMap内部是一个哈希表的数据结构，存储的是键值对，通过key的hash值与数组长度取模计算出value在数组中的位置，出现冲突的对象使用链地址法采用尾部插入的方式组成链表。
哈希表的查找效率是O(1)。

Hash方法 & 扰动函数

上面的这个操作被称为扰动函数，通过这样的计算可以结合高位的特性以此提升hash值的随机性。

  • 因为计算元素位置是通过 (n - 1) & hash 来实现的。例如数组长度为16，它减一后的二进制表示为00001111（前面没画出的都是用0补齐），hash值与这个值取余只能获取到hash值的后面四位（因为数组减一前面都是0，与操作后都是0）。假如有两个hash值，他们的后四位相同，前面17位到21位可能有不同。如果使用直接取余的方式，那么他俩就会发生哈希冲突，但是如果与高16位进行异或操作，那么结果就可能不会发生冲突，因为17到21位不同，异或后的结果也就不一样了，这样增强了随机性也就达到了降低冲突的效果。所以叫做保留了高16位的特性，否则高16位可能根本使用不上。

添加方法 put(K key, V value)

put方法小结：

  • 调用方法时先检查数组是否初始化了，如果没有则指定默认值16初始化数组。
  • 判断数组头是否已经有值了，如果没有则将元素放入即可，否则进行后边判断。
  • 如果数组头的key跟要插入的key一样，则记录这个键值对。
  • 如果数组头是树节点，则要将value作为一个树节点插入到树中。
  • 如果数组头不是树节点，那么就是链表结构，就需要遍历链表。遍历过程中如果找到相同key了，那么直接记录键值对跳出循环，否则在链表尾部插入键值对。
  • 如果有记录的键值对，代表有可以相同的节点，则返回旧值，用新值覆盖旧值。

扩容方法 resize()

HashMap的容量 = 加载因子 * 数组长度，当HashMap中的键值对数量超过这个值的时候，就会触发resize操作。
这个方法概括一下做了两件事情。

  • 将数组扩容为原来的二倍。
  • 将原来的元素重新hash到新的数组上。

小结

  • 判断数组是否为null，数组长度是否为0.
  • 为0代表没有初始化，数组初始化成默认大小，容量赋值为数组默认大小*加载因子。
  • 不为0代表已经初始化了，则将数组和容量的大小通过左移一位变成原来的二倍。
    如果没有初始化，会在第一次调用put方法时调用resize方法，进行默认设置，这是hashmap的延时加载。

现在来看第二件事，怎么将元素重hash到新的数组上。这里的操作十分巧妙，且往下看。

小结

  • 遍历数组，判断数组位置是否为空。如果为空则不需要进行rehash，否则需要hash到新的数组上。
  • 不为空时，判断是链表结构还是红黑树结构。
  • 红黑树结构的话进行红黑树的rehash。
  • 如果是链表的话，通过将原链表上元素的hash值与原数组长度做与操作的结果是0是1分成两个链表。分别放到新数组原位置和原位置+原数组长度位置。

问题

  • 高低位链表是做什么的？
    这就是精巧的地方，自己举几个例子就会发现，通过元素的hash值和与原数组长度与操作的结果如果是0，那么这个hash值与新数组长度取余的话结果也是原来的数组下标。如果结果是1，那么这个hash值与新数组长度取余的结果就是原位置+新数组长度。

删除操作 remove(Object key)

小结：

  • 计算要删除的key在数组中对应位置。
  • 判断数组位置的节点是什么类型的。
  • 红黑树的话，调用红黑树删除方法。
  • 链表的话，找到要删除的节点，将前一个节点的next指向要删除节点的next节点。

获取方法get(Object key)

小结：

  • 通过key的hash值计算出在数组中的位置。
  • 判断头节点的key是否相同，相同则返回。
  • 如果头节点有后续节点，并且判断节点类型。
  • 红黑树节点的话调用红黑树的查找方法。
  • 链表的话遍历链表，找到key相同的节点后返回。

其他

HashTable和HashMap的区别？

  • HashTable是线程安全的，HashMap是线程不安全的。
  • HashTable中的value不能为null，否责会抛出空指针异常。
  • HashTable扩容时，新数组长度为原数组长度的二倍加一，HashMap是原来的二倍。
  • HashTable的扩容方式是遍历数组位值，遍历链表用key的hash值与数组长度直接取余，之后使用头插法插入。HashMap是遍历数组，之后遍历链表生成高低位两个链表，使用的尾插法，之后将两个链表分别放到原位置和原位置加原数组长度位置。
  • HashTable只使用链表，HashMap在链表长度操作8时会将链表转化为红黑树。

ConcurrentHashMap和HashTable选择？

  • 毫无疑问选择ConcurrentHashMap，HashTable是将方式用synchronized锁住，即使同时插入的键值对对应的数组index不同也会阻塞。ConcurrentHashMap（1.8）是通过锁数组index位置的节点来控制并发，这样即使同时插入，但是如果在数组的index不同也可以同时插入，而且ConcurrentHashMap使用了大量的cas操作，降低了线程在不同状态转化的消耗，ConcurrentHashMap源码解析以后会写。

HashMap数组长度为什么必须是2的整数次幂

  • 数组下标通过hash & (n - 1)来进行计算，当数组长度n为2的正数次幂时，n-1结果的二进制全都是1，这样可以使得&操作的结果更加分散。如果n不是2的幂次，那么减1就可能有的位是0，那么与hash值做&操作，无论hash值的对应位是0还是1都会被散列到同一个位置。
  • 因为hashmap确定键值对位置是通过key的hash值与数组长度减一做与操作来确定，只有数组长度是2的整数次幂才能使(hash & (length-1)) == (hash % length)。

HashMap的hash函数为什么要对高16位做异或操作

  • 这个操作叫做扰动函数，因为这样可以结合hash值的高16位的特性，减少低16位相同高16位不相同的key发生碰撞，减少hash冲突概率。
