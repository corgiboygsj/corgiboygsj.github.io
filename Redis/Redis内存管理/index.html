过期键回收策略

由于内存中存在大量的键，如果精准维护过期时间会很耗费cpu资源。因此Redis通过定时删除和惰性删除来解决过期键的淘汰。

定时删除

Redis随机获取部分的键，并且删除其中过期的键值对。根据过期键的比例，回收分为快模式和慢模式。流程如下

（1）首先通过慢模式进行回收，Redis默认每10秒随机获取20个键。
（2）其中如果25%的键是过期的，那么Redis会执行顺序回收，直到获取的20个键中过期的不足25%或者回收时间超过25ms。
（3）如果之前触发了超时，那么在Redis内部事件发生时会进行一次快回收，超时时间是1ms。

惰性删除

惰性删除指的是，当Redis访问键的时候会先查看这个键是否已经过期了。如果过期的话就会把找个键删除并且返回null。

内存淘汰策略

内存淘汰策略指当Redis内存满了的情况下，再次往里面插入键值，就会触发内存淘汰策略。

（1）noeviction
不做处理，只是返回错误。这个是默认策略。

（2）allkeys-random
随机从所有key中删除，直到释放的空间满足新的键值对。

（3）volatile-random
从所有过期的key中删除，直到释放的空间满足新的简直对。

（4）allkeys-lru
从所有的key中按照lru算法，删除最老使用的key直到空间满足。

（5）volatile-lru
从设置了过期时间的key中按照lru算法，删除最老使用的key直到空间满足。

（6）volatile-ttl
删除剩余过期时间最短的key，直到空间满足。
